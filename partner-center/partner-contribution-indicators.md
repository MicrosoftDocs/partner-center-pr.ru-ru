Вы можете видеть ход выполнения PCI на панели мониторинга участия партнера. Данные обновляются на 20-й день каждого месяца с данными за предыдущий месяц. Например, данные за февраль будут опубликованы 20 марта.

## <a name="overview-of-partner-contribution-indicators-for-this-competency"></a>Обзор индикаторов участия партнеров для этой компетенции

Индикаторы участия партнера предоставляют целостную меру влияния партнера на выполнение успешных выполнений клиентов и, таким образом, более подробного различения рынка, когда партнер достиг или удерживает корпоративное Business Applications компетенции для облака.

С помощью PCI партнеры получают баллы за ключевые показатели успеха клиента, такие как успешные развертывания и повышение эффективности использования. Они вознаграждены на продолжение инвестиций и масштабирования своих технических возможностей.

Если партнер достигает определенных уровней влияния в ключевых показателях, то, например, партнер может достичь (или удержать) компетенции для Business Applications в облаке или категории Gold Cloud.

> [!NOTE]
> Дополнительные сведения о преимуществах достижения компетенций см. в статье [компетенции партнеров Майкрософт](https://partner.microsoft.com/membership/competencies).

## <a name="pci-scoring-based-on-seven-key-indicators"></a>Оценка PCI на основе семи ключевых индикаторов

PCI предоставляет комплексное представление о технических возможностях партнера, их возможности по обеспечению успеха и производительности продаж. Чтобы помочь партнерам достичь компетенции корпоративного Business Applications Gold, PCI измеряет производительность в предыдущем году в семи ключевых индикаторах.

*Индикаторы производительности:*

1. **Чистая новая прибыль**: чистая новая прибыль по всем каналам продаж для продуктов Microsoft Dynamics 365. (Дополнительные сведения об этой метрике см. в разделе [определения для чистого нового дохода](partner-contribution-indicators.md#definitions-for-pci-metric-1---net-new-revenue).)

2. **Добавление клиентов в сеть**: чистые клиенты, приобретенные для Dynamics 365 по всем каналам продаж. (Дополнительные сведения об этой метрике см. в разделе [определения для сетевых добавлений клиентов](partner-contribution-indicators.md#definitions-for-pci-metric-2---net-customer-adds).)

*Индикаторы возможностей:*

3. **Функциональные консультанты**. количество человек, занимающихся действующей в настоящее время функциональным консультантом связь с сертификатом для Dynamics 365. (Дополнительные сведения об этой метрике см. в разделе [определения функциональных консультантов](partner-contribution-indicators.md#definitions-for-pci-metric-3---functional-consultants).)

4. **Рост функционального консультанта**: рост количества функциональных партнеров. Есть несколько вариантов получения точек для этого индикатора. Один из вариантов требует, чтобы базовый моментальный снимок для функционального показателя роста функциональности консультанта в PCI стал 1 октября 2020 г. (Дополнительные сведения о всех вариантах получения точек для этой метрики см. в разделе [определения функционального роста консультанта](partner-contribution-indicators.md#definitions-for-pci-metric-4---functional-consultant-growth).)

5. **Разработчики**: количество лиц, занимающихся назначением текущего активного в настоящее время сертификата для Dynamics 365. (Дополнительные сведения об этой метрике см. в разделе [определения для разработчиков](partner-contribution-indicators.md#definitions-for-pci-metric-5---developers).)

*Индикаторы успеха клиента:*

6. **Новые крупные развертывания**: количество рабочих клиентов, которые пересекают пороговое значение высокой загрузки за последние 12 месяцев. (Дополнительные сведения об этой метрике см. в разделе [определения для новых крупных развертываний](partner-contribution-indicators.md#definitions-for-pci-metric-6---new-large-deployments).)

7. **Месячный активный пользователь**: рост числа ежемесячных активных пользователей по всем клиентам за последние 12 месяцев. (Дополнительные сведения об этой метрике см. в разделе [определения для ежемесячного увеличения числа активных пользователей](partner-contribution-indicators.md#definitions-for-pci-metric-7---monthly-active-user-growth).)

## <a name="thresholds-for-each-pci-metric-for-this-competency"></a>Пороговые значения для каждой метрики PCI для этой компетенции

Каждая метрика PCI имеет пороговые значения, заданные корпорацией Майкрософт, которые могут принести вашей компании максимальные баллы. Эти пороговые значения отличаются в отношении взаимодействия с клиентами и финансового пути и путей операций, используемых для достижения компетенции облачных Business Applications. Пороговые значения также отличаются для партнеров, работающих на развитых рынках, и для партнеров, работающих исключительно на развивающихся рынках. **Разработанная** или **разрабатываемая** классификация рынка основана на [определениях Microsoft Partner Network](https://assetsprod.microsoft.com/mpn/mpn-developed-and-developing-countries.pdf).

Обратите внимание, что 4 метрики измеряются на основе расположения, в котором партнер выполняет бизнес, а не в том месте, где зарегистрировано расположение MPN. Это означает, что партнер, зарегистрированный на рынке разработки, который имеет клиентов из разработанного рынка, должен соответствовать рейтингу PCI для разработанных стран.

В таблице ниже показано, какое расположение применяется метрикой.

| Имя метрики | Расположение |
|:-------|:--------|
| Новый чистый доход | Расположение клиента |
| Добавление клиента NET | Расположение клиента |
| Функциональный консультант | Расположение партнера |
| Developer | Расположение партнера |
| Рост для консультантов по функциональным возможностям | Расположение партнера |

Баллы по каждой метрике подписываются для получения уникального рейтинга PCI для взаимодействия с клиентами и финансовыми операциями. Этот показатель будет обновляться ежемесячно.

## <a name="prerequisites-to-receive-pci-credit-for-a-customer"></a>Необходимые условия для получения кредита PCI для клиента

Некоторые индикаторы участия партнера позволяют получить кредит для определенного клиента, если у вас есть определенные связи между партнерами и клиентами. Ниже перечислены индикаторы и метрики PCI, на которые влияют следующие компоненты.

- Новый чистый доход
- Чистое количество добавленных клиентов
- Новые развертывания
- Ежемесячный рост по активным пользователям

Вы можете получить кредит для этих метрик PCI для определенного клиента, если у вас есть одна из следующих связей "партнер-клиент":
- Вы являетесь поставщиком облачных решений (CSP) для платной подписки, относящейся к клиенту клиента.
- Вы являетесь партнером по заявке на запись (КПОР) для платной подписки, относящейся к клиенту клиента.
- Вы являетесь цифровым партнером записи (DPOR) для платной подписки, относящейся к клиенту клиента.

Партнеры становятся КПОР, если они установили связь с клиентом на уровне подписки с помощью стимула помощника по активации служб Dynamics 365 Online Services (OSA) или использования Dynamics 365 Online Services (осу). Дополнительные сведения см. в разделе [понятие КПОР Associations](incentives-customer-associations-intro.md).

Партнер получает статус DPOR, если он установил связь с клиентом на уровне подписки в рамках продукта.

> [!NOTE]
> Если для данной подписки клиента существует и КПОР, и DPOR, КПОР получит кредит для соответствующих метрик PCI, связанных с этой подпиской. Если ассоциация была выполнена через OSA (заявка на выручку), КПОР получает кредит для новых доходов и клиентов .NET. Если ассоциация была выполнена через осу (заявка на использование), КПОР получает кредит для новых развертываний и ежемесячного роста активного пользователя. Утверждения предварительной продажи не подходят для PCI.

### <a name="definitions-for-pci-metric-1---net-new-revenue"></a>Определения для #1 "Метрика PCI" — новый доход

Чистая новая прибыль — это [индикатор производительности](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Баллы этого индикатора рассчитываются для компетенции облачной Business Applications следующим образом:

- Чистая новая прибыль определяется как чистая новая Выручка по всем каналам продаж для квалификации продуктов Dynamics 365.
- Чистая новая прибыль суммируется за последний 12-месячный период с максимальным значением 10 очков.
- Баллы за чистый новый доход рассчитываются как счета за вычетом кредитов и суммируются в течение 12 месяцев.
- Чистая новая прибыль вычисляется на основе следующих, подходящих продуктов:
  - Продукты, подходящие для сотрудников, включают в себя: продажи, Обслуживание клиентов, службу полей, автоматизацию службы проектов, маркетинг, продажи Microsoft relationship, план 1, менеджер по продажам и корпоративный отдел.
  - Продукты, подходящие для финансового и Operations: унифицированные операции, финансы, управление логистическими цепочками, розничная торговля, план 2, Смешанная реальность, специалист по бизнесу и Группа специалистов.

- К каналам продаж относятся Соглашение Enterprise (EA), поставщик облачных решений (CSP) и Web Direct.
- Только для Соглашение Enterprise прибыли: этот доход включает доход от первого года соглашения. Повторные доходы за второй и третий годы не включаются.

### <a name="definitions-for-pci-metric-2---net-customer-adds"></a>Определения для #2 "Метрика PCI" — Добавление клиентов в сеть

Добавление клиентов NET — это [индикатор производительности](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Баллы этого индикатора рассчитываются для компетенции облачной Business Applications следующим образом:

- В течение 12-месячного числа добавлений клиентов суммируется с максимальной возможной заработкой 10 очков. 

- Клиенты .NET добавляются по всем каналам продаж. 

- Баллы за добавление в сеть клиентов рассчитываются ежемесячно по мере того, как клиенты добавляются за вычетом потерянных клиентов и суммируются в течение 12 месяцев. 

- В этих условиях определяется **Клиентская** часть: когда у клиента нет платных мест для всех подписок в течение прошлого месяца, но в текущем месяце имеется по крайней мере одно Платное место.

- **Потерянный клиент** определяется в следующих условиях: когда у клиента есть по крайней мере одно Платное место для всех подписок в течение прошлого месяца, **а** также нет платных рабочих мест в текущем месяце.

- Максимальное и частичное количество очков может быть получено для клиентов, добавленных в сеть. Ниже приведены требования для получения максимальных очков для данного компетенции:

  | Для этой пути | Число добавлений клиентов в течение максимум 10 очков |
  |---------------------|----------------------------|
  | Customer Engagement | – 10 клиентов добавляет (для партнеров на *развитых* рынках);</br>-5 NET Добавление клиентов (для партнеров по *разработке* рынков)  |
  | Finance and Operations | – 3 NET добавляет (для партнеров на *развитых* рынках);</br>— 1 Добавление нового клиента (для партнеров в *разработке* рынков); |

  > [!NOTE]
  > Дополнительные сведения об разработанных и развивающихся рынках см. в этих [определениях Microsoft Partner Network](https://assetsprod.microsoft.com/mpn/mpn-developed-and-developing-countries.pdf).

- К каналам продаж относятся Соглашение Enterprise (EA), поставщик облачных решений (CSP) и Web Direct.

- Для нескольких партнеров и одного клиента или клиента, работающих непосредственно с корпорацией Майкрософт, существуют различные вычисления для **клиентов** , а **Клиенты теряются** в случаях, когда несколько партнеров связаны с одним клиентом по разным подпискам или в случаях, когда клиент уже может работать непосредственно с корпорацией Майкрософт. Эти вычисления отображаются ниже.

  **Для нескольких партнеров, имеющих одного клиента, разные подписки:**

- Клиент добавлен в первый раз:
    - Все партнеры, связанные с клиентом, получат кредит для нового клиента, добавляемого в месяц, в который добавляется клиент.
    - Последующие партнеры, связывающие с одним и тем же клиентом с помощью старых или новых подписок, не получат новый клиент добавить кредит для этого клиента.
- Клиент теряется: все связанные партнеры, которые в течение предыдущих 12 месяцев получили кредит за добавлением клиента, потеряют кредит в месяце, когда клиент будет потерян.
- Клиент, который ранее был потерян, добавлен назад: все партнеры, связанные с клиентом, получат кредит для нового клиента, добавляемого в месяц, в который добавляется клиент.
  
  **Когда клиент уже работает непосредственно с корпорацией Майкрософт:**

   - Если клиент уже работает непосредственно с корпорацией Майкрософт и соответствует новому критерию добавления клиента, партнеры, связывающиеся с тем же клиентом в дальнейшем, с помощью старых или новых подписок, не получат новый клиент, чтобы добавить кредит для этого клиента.

### <a name="definitions-for-pci-metric-3---functional-consultants"></a>Определения для #3 метрик PCI — функциональные консультанты

Функциональные консультанты — это [индикатор возможностей](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Баллы этого индикатора рассчитываются для компетенции облачной Business Applications следующим образом:

  - МБ — 210, МБ — 220, МБ — 230, МБ — 240 или MB6-898 \* \* .

  **и** один из следующих экзаменов:

Функциональный рост консультанта — это [индикатор возможностей](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Баллы этого индикатора рассчитываются для компетенции облачной Business Applications следующим образом:

### <a name="definitions-for-pci-metric-6---new-large-deployments"></a>Определения для #6 метрик PCI — новые крупные развертывания

Новые крупные развертывания — это [индикатор успеха клиента](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Баллы этого индикатора рассчитываются для компетенции облачной Business Applications следующим образом:

- Новые крупные развертывания суммируются в течение 12-месячного периода до 20 доступных очков.

- Новые крупные развертывания определяются по-разному для взаимодействия клиентов с финансами и эксплуатации, а для партнеров, *работающих в развивающихся* рынках и *развивающихся* рынках:

  - Для участия клиентов новые крупные развертывания определяются как количество рабочих экземпляров в клиенте, которые получили **250** ежемесячных активных пользователей (для разработанных партнеров) и **150** ежемесячных активных пользователей (для разработки партнеров по маркетингу).

  - Для финансов и операций новые крупные развертывания определяются как количество рабочих экземпляров в клиенте, которые получили **100** ежемесячных активных пользователей (для разработанных потребительских партнеров) и **50** ежемесячных активных пользователей (для разработки рыночных партнеров).

  - Это пороговое значение пересекается не менее одного раза в течение последующих 11 месяцев.

- Активные пользователи будут учитываться на уровне клиента и суммироваться только по платным подпискам в рамках клиента.

- Активные пользователи за месяц вычисляются в конце каждого месяца.

- Партнеры связаны с клиентами на уровне подписки. Подписки сопоставляются со следующими, подходящими для участия в продуктах по взаимодействию с клиентами или с финансовыми и операционными продуктами.

### <a name="definitions-for-pci-metric-7---monthly-active-user-growth"></a>Определения для метрики PCI #7 — месячный активный рост пользователей

Ежемесячное увеличение числа активных пользователей — это [индикатор успеха клиента](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Баллы этого индикатора рассчитываются для компетенции облачной Business Applications следующим образом:

- Месячный активный рост пользователей определяется как рост числа активных пользователей в рамках всей клиентской базы за последние 12 месяцев. 

- Активные пользователи будут учитываться на уровне клиента и суммироваться только по платным подпискам в рамках клиента.

- Индикатор месячного увеличения активного пользователя Сравнивает текущий месяц с базовым планом с 12 месяцев назад. доступно 25 максимальных очков.

- Активные пользователи за месяц вычисляются в конце каждого месяца.

- Ежемесячное увеличение числа активных пользователей определяется по-разному для взаимодействия с клиентами и финансовыми и операционными учетными данными, а для партнеров, работающих на *развитых* рынках и для *разработки* рынков:

  - Чтобы участие клиентов было допустимым для баллов, разработанные партнеры по маркетингу должны иметь базовые показатели по крайней мере **250** месячных активных пользователей, а разработка партнеров рынка должна иметь базовую сумму **150** ежемесячных активных пользователей по всей клиентской базе данных 12 месяцев назад.

  - Чтобы финансы и операции были доступны для точек, разработанные партнеры рынка должны иметь базовые показатели по крайней мере **100** месячных активных пользователей, а разработка потребительских партнеров должна иметь базовую сумму **50** ежемесячных активных пользователей по всему клиенту в 12 месяцах назад.
  
  (Дополнительные сведения об разработанных и развивающихся рынках см. в этих [определениях Microsoft Partner Network](https://assetsprod.microsoft.com/mpn/mpn-developed-and-developing-countries.pdf).

- Партнеры связаны с клиентами на уровне подписки. Подписки сопоставляются с подходящими продуктами по взаимодействию с клиентами или с учетом финансовых и операционных продуктов.

- Все партнеры, связанные с любой платной подпиской в клиенте, будут получать кредит для всех клиентов или финансовых операций, а также выполнять операции ежемесячных активных пользователей, суммированных по всем платным подпискам в этом клиенте.
