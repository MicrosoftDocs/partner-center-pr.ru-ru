## <a name="overview-of-partner-contribution-indicators"></a>Обзор индикаторов участия партнера

Индикаторы участия партнера предоставляют целостную меру влияния партнера на выполнение успешных результатов клиентов и, таким образом, более рентабельность рынка, когда партнер достиг и обслуживает компетенцию Cloud Business Applications (CBA) MPN.

С помощью PCI партнеры получают баллы за ключевые показатели успеха клиентов, такие как успешные развертывания и повышение уровня использования, и возросли на то, чтобы продолжить вкладывать и масштабировать свои технические возможности.

## <a name="pci-scoring-based-on-seven-key-indicators"></a>Оценка PCI на основе семи ключевых индикаторов

PCI предоставляет комплексное представление о технических возможностях партнера, их возможности по обеспечению успеха и производительности продаж. PCI измеряет производительность прошлого года компании в семи ключевых индикаторах:

*Индикаторы производительности:*

1. **Чистая новая прибыль** : чистая новая прибыль по всем каналам продаж для продуктов Microsoft Dynamics 365. (Дополнительные сведения об этой метрике см. в разделе [определения для чистого нового дохода](partner-contribution-indicators.md#definitions-for-pci-metric-1---net-new-revenue).)

2. **Добавление клиентов в сеть** : чистые клиенты, приобретенные для Dynamics 365 по всем каналам продаж. (Дополнительные сведения об этой метрике см. в разделе [определения для сетевых добавлений клиентов](partner-contribution-indicators.md#definitions-for-pci-metric-2---net-customer-adds).)

*Индикаторы возможностей:*

3. **Функциональные консультанты**. количество человек, занимающихся действующей в настоящее время функциональным консультантом связь с сертификатом для Dynamics 365. (Дополнительные сведения об этой метрике см. в разделе [определения функциональных консультантов](partner-contribution-indicators.md#definitions-for-pci-metric-3---functional-consultants).)

4. **Рост функционального консультанта** : рост количества функциональных партнеров.  Базовый моментальный снимок показателя роста функциональности функционального консультанта в PCI теперь будет выполнен 1 октября 2020 г. (Дополнительные сведения об этой метрике см. в разделе [определения функционального роста консультанта](partner-contribution-indicators.md#definitions-for-pci-metric-4---functional-consultant-growth).)

5. **Разработчики** : количество лиц, занимающихся назначением текущего активного в настоящее время сертификата для Dynamics 365. (Дополнительные сведения об этой метрике см. в разделе [определения для разработчиков](partner-contribution-indicators.md#definitions-for-pci-metric-5---developers).)

*Индикаторы успеха клиента:*

6. **Новые крупные развертывания** : количество рабочих клиентов, которые пересекают пороговое значение высокой загрузки за последние 12 месяцев. (Дополнительные сведения об этой метрике см. в разделе [определения для новых крупных развертываний](partner-contribution-indicators.md#definitions-for-pci-metric-6---new-large-deployments).)

7. **Месячный активный пользователь** : рост числа ежемесячных активных пользователей по всем клиентам за последние 12 месяцев. (Дополнительные сведения об этой метрике см. в разделе [определения для ежемесячного увеличения числа активных пользователей](partner-contribution-indicators.md#definitions-for-pci-metric-7---monthly-active-user-growth).)

## <a name="thresholds-for-each-pci-metric"></a>Пороговые значения для каждой метрики PCI

Каждая метрика PCI имеет пороговые значения, заданные корпорацией Майкрософт, которые могут принести вашей компании максимальные баллы. Эти пороговые значения отличаются в отношении взаимодействия с клиентами, финансового пути и операций для достижения компетенции облачной Business Applications. Пороговые значения также отличаются для партнеров, работающих на развитых рынках, и для партнеров, работающих исключительно на развивающихся рынках. **Разработанная** или **разрабатываемая** классификация рынка основана на [определениях Microsoft Partner Network](https://assetsprod.microsoft.com/mpn/mpn-developed-and-developing-countries.pdf).

Баллы по каждой метрике подписываются для получения уникального рейтинга PCI для взаимодействия с клиентами и финансовыми операциями. Этот показатель будет обновляться ежемесячно.

Партнеры могут получить максимальное количество баллов для заданной метрики, если они достигли или превысили пороговые значения, указанные ниже. Частичные баллы можно получить для всех метрик, за исключением функционального роста консультанта.

### <a name="definitions-for-pci-metric-1---net-new-revenue"></a>Определения для #1 "Метрика PCI" — новый доход

Чистая новая прибыль — это индикатор производительности и один из семи ключевых показателей участия партнера, [описанных выше в этой статье](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Ниже приведены дополнительные сведения о том, как определяется чистый доход и как рассчитываются его баллы.

- Чистая новая прибыль определяется как чистая новая Выручка по всем каналам продаж для квалификации продуктов Dynamics 365.
- Чистая новая прибыль суммируется за последний 12-месячный период с максимальным значением 10 очков.
- Баллы за чистый новый доход рассчитываются как счета за вычетом кредитов и суммируются в течение 12 месяцев.
- Чистая новая прибыль вычисляется на основе следующих, подходящих продуктов:

   - Продукты, подходящие для сотрудников, включают в себя: продажи, Обслуживание клиентов, службу полей, автоматизацию службы проектов, маркетинг, продажи Microsoft relationship, план 1, менеджер по продажам и корпоративный отдел.
   - Продукты, подходящие для финансового и Operations: унифицированные операции, финансы, управление логистическими цепочками, розничная торговля, план 2, Смешанная реальность, специалист по бизнесу и Группа специалистов.

- К каналам продаж относятся Соглашение Enterprise (EA), поставщик облачных решений (CSP) и Web Direct.
- Только для Соглашение Enterprise прибыли: этот доход включает доход от первого года соглашения. Повторные доходы за второй и третий годы не включаются.

### <a name="definitions-for-pci-metric-2---net-customer-adds"></a>Определения для #2 "Метрика PCI" — Добавление клиентов в сеть

Добавление клиентов NET — это индикатор производительности и один из семи ключевых показателей участия партнера, [описанных выше в этой статье](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Ниже приведены дополнительные сведения о том, как определяются сетевые добавления клиентов и как рассчитываются точки этой категории:

- В течение 12-месячного числа добавлений клиентов суммируется с максимальным количеством возможных 10 очков.

- Клиенты .NET добавляются по всем каналам продаж. 

- Баллы за добавление в сеть клиентов рассчитываются ежемесячно по мере того, как клиенты добавляются за вычетом потерянных клиентов и суммируются в течение 12 месяцев. 

- В этих условиях определяется **Клиентская** часть: когда у клиента нет платных мест для всех подписок в течение прошлого месяца, но в текущем месяце имеется по крайней мере одно Платное место.

- **Потерянный клиент** определяется в следующих условиях: когда у клиента есть по крайней мере одно Платное место для всех подписок в течение прошлого месяца, **а** также нет платных рабочих мест в текущем месяце.

- К каналам продаж относятся Соглашение Enterprise (EA), поставщик облачных решений (CSP) и Web Direct.

- Для нескольких партнеров и одного клиента или клиента, работающих непосредственно с корпорацией Майкрософт, существуют различные вычисления для **клиентов** , а **Клиенты теряются** в случаях, когда несколько партнеров связаны с одним клиентом по разным подпискам или в случаях, когда клиент уже может работать непосредственно с корпорацией Майкрософт. Эти вычисления отображаются ниже.

  **Для нескольких партнеров, имеющих одного клиента, разные подписки:**

  - Все партнеры, связанные с клиентом, получат кредит для нового клиента, добавляемого в месяц, в который добавляется клиент.
  
  - Последующие партнеры, связывающие с одним и тем же клиентом с помощью старых или новых подписок, не получат новый клиент добавить кредит для этого клиента.
  
  - Все партнеры, связанные с клиентом, будут потеряют кредит для нового клиента, добавленного в месяц, когда клиент будет потерян.
  
  - Любой партнер, сопоставленный с тем же клиентом в дальнейшем, через старые или новые подписки, получит новый клиент добавить кредит для этого клиента.
  
  **Когда клиент уже работает непосредственно с корпорацией Майкрософт:**

   - Если клиент уже работает непосредственно с корпорацией Майкрософт и соответствует новому критерию добавления клиента, партнеры, связывающиеся с тем же клиентом в дальнейшем, с помощью старых или новых подписок, не получат новый клиент, чтобы добавить кредит для этого клиента.

### <a name="definitions-for-pci-metric-3---functional-consultants"></a>Определения для #3 метрик PCI — функциональные консультанты

Функциональные консультанты — это индикатор возможностей и один из семи ключевых показателей участия партнера, [описанных выше в этой статье](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Ниже приведены дополнительные сведения о том, как определяется индикатор функциональных консультантов и как рассчитываются точки этой категории.

- Функциональные точки индикатора функционального консультанта основаны на моментальном снимке текущего месяца с максимальной возможной заработкой 15 очков.

- Функциональные консультанты определяются по мере того, как число людей, занимающихся назначением текущего активного интерактивного консультанта, связывается с сертификацией, например, сертификация по функциональным консультантам по взаимодействию с клиентами или финансовым консультантом.

- Для сертификации функционального консультанта по взаимодействию с клиентами требуется: Мб-200 **и** один из следующих: мб-210, мб-220, мб-230, mb-240 или MB6-898.

- Для сертификации "Финансы" и "эксплуатация по Operations" необходимо: МБ-300 **и** один из следующих: мб-310, мб-320, мб-330, MB6-897 или MB6-898.

### <a name="definitions-for-pci-metric-4---functional-consultant-growth"></a>Определения для #4 метрик PCI — функциональный рост консультанта

Функциональный рост консультанта является индикатором возможностей и одним из семи ключевых показателей участия партнера, [описанных выше в этой статье](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Ниже приведены дополнительные сведения о том, как определяется индикатор роста функционального консультанта и как рассчитываются точки этой категории:

- Функциональный рост консультанта определяется как рост числа сертифицированных взаимосвязи с функциональным консультантом с момента создания базового моментального снимка. Дата создания базового моментального снимка будет составлять 1 октября 2020 г.

- Функциональные точки индикатора роста функционального консультанта основаны на снимке текущего месяца по сравнению с базовым планом 1 октября 2020 г. Этот индикатор позволяет партнерам получить от 0 до 15 баллов.

- Базовый моментальный снимок будет впоследствии сброшен в следующей облачной Business Applications дату продления компетенции.

- Чтобы иметь право на точки, необходимо иметь не менее **15** подходящих партнерских консультантов для разработанных рыночных партнеров и **5** для разработки рыночных партнеров при создании базового моментального снимка.

- **Частичные точки не вычисляются.** Увеличение размера базового моментального снимка должно соответствовать или превышать пороговые значения, указанные в [таблице 3. Рост функционального консультанта](partner-contribution-indicators.md#table-3-functional-consultant-growth).

### <a name="definitions-for-pci-metric-5---developers"></a>Определения для #5 метрик PCI — разработчики

Разработчики являются индикатором возможностей и одним из семи ключевых показателей участия партнера, [описанных выше в этой статье](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Ниже приведены дополнительные сведения о том, как определяется индикатор разработчиков и как рассчитываются точки этой категории:

- Разработчики определяются как количество лиц, занимающихся сертификацией, которые в настоящее время активны для разработчиков.

- Точки для индикатора разработчика основаны на моментальном снимке текущего месяца, для которого доступно не более 5 очков.

- Разработчики по взаимодействию с клиентами получают кредит, когда они пройдут оба Мб – 200 **и** один из следующих: mb-400 или MB2-716.

- Финансовые и операционные разработчики получают кредит при передаче как МБ-300 **, так и** одного из следующих: mb-500 или MB6-894.

### <a name="definitions-for-pci-metric-6---new-large-deployments"></a>Определения для #6 метрик PCI — новые крупные развертывания

Новые крупные развертывания — это индикатор успеха клиента и один из семи ключевых показателей участия партнера, [описанных выше в этой статье](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Ниже приведены дополнительные сведения о том, как определяется индикатор нового крупного развертывания и как рассчитываются точки категории.

- Новые крупные развертывания суммируются в течение 12-месячного периода до 20 доступных очков.

- Новые крупные развертывания определяются по-разному для взаимодействия с клиентами и финансовыми операциями.

  - Для участия клиентов новые крупные развертывания определяются как количество рабочих экземпляров в клиенте, которые получили **250** ежемесячных активных пользователей (для разработанных партнеров) и **150** ежемесячных активных пользователей (для разработки партнеров по маркетингу).

   - Для финансов и операций новые крупные развертывания определяются как количество рабочих экземпляров в клиенте, которые получили **100** ежемесячных активных пользователей (для разработанных потребительских партнеров) и **50** ежемесячных активных пользователей (для разработки рыночных партнеров).

- Предполагается, что клиент пересекает пороговое значение активного пользователя, если клиент был: 

  - Ниже указанного порога 12 месяцев назад *и*
  - Это пороговое значение пересекается не менее одного раза в течение последующих 11 месяцев.

- Активные пользователи будут учитываться на уровне клиента и суммироваться только по платным подпискам в рамках клиента.
- Активные пользователи за месяц вычисляются в конце каждого месяца.
- Партнеры связаны с клиентами на уровне подписки. Подписки сопоставляются с продуктами по взаимодействию с клиентами или с финансовыми и операционными продуктами.
- Все партнеры, связанные с любой платной подпиской в клиенте, получат кредит для всех рабочих экземпляров клиентов, а также для финансовых и производственных операций в этом клиенте.

### <a name="definitions-for-pci-metric-7---monthly-active-user-growth"></a>Определения для метрики PCI #7 — месячный активный рост пользователей

Ежемесячное увеличение числа активных пользователей — это индикатор успеха клиента и один из семи ключевых показателей участия партнера, [описанных выше в этой статье](partner-contribution-indicators.md#pci-scoring-based-on-seven-key-indicators). Ниже приведены дополнительные сведения о том, как определяется ежемесячное число активных пользователей и как рассчитываются точки этой категории.

- Месячный активный рост пользователей определяется как рост числа активных пользователей в рамках всей клиентской базы за последние 12 месяцев. 

- Активные пользователи будут учитываться на уровне клиента и суммироваться только по платным подпискам в рамках клиента.

- Индикатор месячного увеличения активного пользователя Сравнивает текущий месяц с базовым планом с 12 месяцев назад. доступно 25 максимальных очков.

- Активные пользователи за месяц вычисляются в конце каждого месяца.

- Чтобы участие клиентов было допустимым для баллов, разработанные партнеры по маркетингу должны иметь базовые показатели по крайней мере **250** месячных активных пользователей, а разработка партнеров рынка должна иметь базовую сумму **150** ежемесячных активных пользователей по всей клиентской базе данных 12 месяцев назад. 

- Чтобы финансы и операции были доступны для точек, разработанные партнеры рынка должны иметь базовые показатели по крайней мере **100** месячных активных пользователей, а разработка потребительских партнеров должна иметь базовую сумму **50** ежемесячных активных пользователей по всему клиенту в 12 месяцах назад. 

- Партнеры связаны с клиентами на уровне подписки. Подписки сопоставляются с участием клиентов или с финансовыми и операционными продуктами.

- Все партнеры, связанные с любой платной подпиской в клиенте, будут получать кредит для всех клиентов или финансовых операций, а также выполнять операции ежемесячных активных пользователей, суммированных по всем платным подпискам в этом клиенте.

## <a name="customer-to-partner-association"></a>Связь между партнером и клиентами

Вы получаете баллы за чистую новую прибыль, NET-Добавление клиентов, новые крупные развертывания и ежемесячное увеличение активного пользователя для данного клиента в пределах определения метрики, только если вы являетесь поставщиком облачных решений (CSP), партнером по заявке на запись (КПОР) или цифровым партнером записи (DPOR) для платной подписки, входящей в клиент клиента.

Партнеры становятся КПОР, если они установили связь с клиентом на уровне подписки с помощью стимула [помощника по активации служб dynamics 365 Online Services](https://support.microsoft.com/help/4501560/online-services-advisor-osa-sell-incentives-faq) (OSA) или [использования Dynamics 365 Online Services](https://support.microsoft.com/help/4489988/online-services-usage-osu-incentives-faq) (осу).

Партнер получает статус DPOR, если он установил связь с клиентом на уровне подписки в рамках продукта.

Если для данной подписки существует и КПОР, и DPOR, КПОР получит кредит для соответствующих метрик PCI, связанных с этой подпиской. КПОР получает кредит для нового дохода от сети, а NET-клиент добавляет, если связь была выполнена через OSA. Если ассоциация была выполнена через осу, КПОР получает кредит для новых крупных развертываний и ежемесячного активного роста пользователей.

## <a name="how-to-access-and-read-the-charts"></a>Доступ к диаграммам и их чтение

В этом разделе описывается, как получить доступ к индикаторам участия партнера из панели мониторинга центра партнеров и как читать связанные диаграммы.

### <a name="access-partner-contribution-indicators"></a>Доступ к индикаторам участия партнера

Чтобы просмотреть диаграммы и ход выполнения ваших партнеров, выполните следующие действия.

1. Выполните вход в [Панель мониторинга](https://partner.microsoft.com/dashboard) Центра партнеров.

2. В меню навигации выберите **CSP** , а затем щелкните **индикаторы участия партнера**. 

   Вы должны увидеть панель мониторинга "Индикаторы участия партнера" со связанными диаграммами.

   > [!NOTE]
   > Если эти параметры не отображаются, попробуйте выбрать **MPN** в меню, а затем — " **индикаторы участия партнера** ". Если вы по-прежнему не можете найти эти параметры, возможно, в центре партнеров не назначена соответствующая роль пользователя. Обратитесь за помощью к администратору глобального администратора или MPN партнеру центра партнеров.
