# <a name="customize-the-out-of-box-experience-for-a-device-with-windows-autopilot-profiles"></a>Настройка готового интерфейса для устройства с профилями Windows "Автопилот"

**Относится к**

- Партнеры с прямым выставлением счетов, косвенные поставщики и непрямые торговые посредники

**Соответствующие роли**

- Агент по администрированию
- Глобальный администратор
- Агент по продажам
- Администратор управления пользователями

Если вы управляете клиентскими устройствами, может потребоваться настроить предварительный запуск (OOBE) для пользователей клиента. Вы можете предварительно настроить новые устройства с помощью профилей Windows "Автопилот", прежде чем доставлять устройства клиентам и применять новые профили к устройствам, которые клиенты уже приобрели. 

Обратите внимание, что изготовители оборудования начали включать метку доставки за пределы поля устройства автопилота, которое показывает **идентификатор ключа продукта (пкид)** устройства.  Этот одномерный доступный для чтения штрих-код предоставляет подчиненным партнерам способ регистрации устройств для автопилота без распаковки устройств и сбора идентификатора устройства с помощью альтернативных средств.

В этой статье объясняется, как создавать и применять профили автопилота к устройствам в центре партнеров.

Если вы еще не знакомы с автопилотом, ознакомьтесь со сведениями в следующих статьях:

- [Общие сведения об автопилоте Windows](https://docs.microsoft.com/windows/deployment/windows-10-auto-pilot)
- [Справочное руководство по развертыванию по для автопилота](https://assetsprod.microsoft.com/autopilot-deployment-program-reference-guide-csp.docx)  

## <a name="overview"></a>Обзор

Функция автопилота Windows в центре партнеров позволяет создавать пользовательские профили для применения к устройствам клиентов. На момент публикации этой статьи были доступны следующие параметры профиля:

- Пропустить параметры конфиденциальности. Этот необязательный параметр профиля автопилота позволяет организациям не запрашивать сведения о параметрах конфиденциальности во время процесса OOBE.

- Отключить создание учетной записи локального администратора на устройстве. Организации могут определить, должен ли пользователь настроить устройство для доступа администратора после завершения процесса.

- Автоматически настроить устройство для работы или учебного заведения. Все устройства, зарегистрированные с помощью автопилота, будут автоматически считаться рабочими или учебными устройствами, поэтому этот вопрос не будет запрашиваться во время процесса OOBE.

- Пропускать страницы настройки регистрации Кортаны, OneDrive и OEM. Все устройства, зарегистрированные с помощью автопилота, автоматически пропускают эти страницы во время готового процесса (OOBE).

- Пропустить лицензионное соглашение (EULA). Начиная с Windows 10 версии 1709, организации могут решить пропустить страницу лицензионного соглашения, представленную в процессе OOBE. Важные сведения о пропуске страницы лицензионного соглашения во время установки Windows см. в статье об [отклонении лицензионного соглашения Windows для автопилота](#windows-autopilot-eula-dismissal) .

Применяются следующие разрешения и ограничения на управление профилями и устройствами.

- Партнеры CSP могут продолжать управлять профилями автопилота для существующих клиентов, которым назначены связи через торгового посредника, даже если клиенты удалили права делегированного администрирования партнера.

- Вы можете управлять существующими устройствами для добавленных клиентов.

- Вы не можете управлять устройствами, отправленными клиентом в Microsoft Store для бизнеса или на портале Microsoft Intune.

## <a name="create-and-manage-autopilot-profiles-in-partner-center"></a>Создание профилей автопилотного развертывания и управление ими в центре партнеров

В центре партнеров можно создать профили развертывания Windows для автопилота и применить их к устройствам.

>[!NOTE]
>Только агенты администрирования могут создавать и применять профили.

### <a name="create-a-new-autopilot-profile"></a>Создать новый профиль для автопилота

1. В меню Центр партнеров выберите **Клиенты** , а затем выберите клиента, для которого создается профиль автопилота.

2. На странице сведения о клиенте выберите **устройства**.

3. В разделе **профили Windows для автопилота** выберите **Добавить новый профиль**.

4. Введите имя и описание профиля, а затем настройте параметры OOBE. Выберите один из следующих типов.  

   - Пропустить параметры конфиденциальности в программе установки

   - Отключение учетной записи локального администратора при настройке
  
   - Автоматический пропуск страниц при настройке<br>
        (В том числе *автоматически выберите Настройка для рабочей или учебной* программы и *пропустите страницы настройки регистрации кортаны, OneDrive и OEM*).
  
   - Пропустить лицензионное соглашение с конечным пользователем (EULA)<br> 
       >[!IMPORTANT] 
       >Важные сведения о пропуске страницы лицензионного соглашения во время установки Windows см. в статье об [отклонении лицензионного соглашения Windows для автопилота](#windows-autopilot-eula-dismissal) .

5. По завершении нажмите кнопку **Отправить**.

### <a name="apply-an-autopilot-profile-to-customer-devices"></a>Применение профиля автопилота для клиентских устройств

>[!NOTE]
>В приведенных ниже инструкциях предполагается, что вы уже добавили устройства клиента в центр партнеров и можете получить доступ к списку устройств. Если вы еще не добавили устройства клиента, следуйте инструкциям в разделе [Добавление устройств в учетную запись клиента](#add-devices-to-a-customers-account) , а затем выполните следующие действия.

После создания профиля автопилота для клиента его можно применить к устройствам клиента.

1. В меню Центр партнеров выберите **Клиенты** , а затем выберите клиента, для которого был создан профиль автопилота.

2. На странице сведения о клиенте выберите **устройства**.

3. В разделе **применить профили к устройствам** выберите устройства или группы устройств, в которые необходимо добавить профили, а затем нажмите кнопку **Применить профиль**. Только что примененный профиль появится в столбце **профиль** .

4. Выполните следующие действия, чтобы убедиться, что профиль будет успешно применен к устройству.

    a.  Подключите устройство к сети и включите его.

    b.  Убедитесь, что отображаются соответствующие экраны OOBE (если они должны отображаться).

    c.  Когда процесс OOBE останавливается, верните устройство к заводским параметрам по умолчанию, чтобы подготовить его для нового пользователя.

### <a name="remove-an-autopilot-profile-from-a-customers-device"></a>Удаление профиля автопилота с устройства клиента

1. В меню Центр партнеров выберите **Клиенты** , а затем выберите клиента, для которого был создан профиль автопилота.

2. На странице сведения о клиенте выберите **устройства**.

3. В разделе **применить профили к устройствам** выберите устройства, из которых нужно удалить профиль, а затем щелкните **удалить профиль**.

   >[!NOTE]
   >Удаление профиля с устройства не приводит к удалению профиля из списка. Если вы хотите удалить профиль, следуйте инструкциям в разделе [обновление или удаление профиля автопилота](#update-or-delete-an-autopilot-profile).

### <a name="update-or-delete-an-autopilot-profile"></a>Обновление или удаление профиля автопилота

Если клиент хочет изменить готовый интерфейс после отправки им устройств, можно изменить профиль в центре партнеров.

Когда устройство клиента подключается к Интернету, оно скачивает последнюю версию профиля во время процесса OOBE. Кроме того, каждый раз, когда клиент восстанавливает заводские параметры по умолчанию, устройство снова загрузит последнюю версию профиля во время процесса OOBE.

1. В меню Центр партнеров выберите **Клиенты** , а затем выберите клиента, желающего изменить профиль автопилота.

2. На странице сведения о клиенте выберите **устройства**.

3. В разделе **профили Windows для автопилота** выберите профиль, который необходимо обновить. Внесите необходимые изменения и нажмите кнопку **Отправить**.

Чтобы удалить этот профиль, выберите **удалить профиль** в правом верхнем углу страницы.

### <a name="add-devices-to-a-customers-account"></a>Добавление устройств в учетную запись клиента

>[!NOTE]
>Агенты продаж и администраторы могут добавлять устройства в учетную запись клиента.

Перед применением пользовательских профилей для автопилота к устройствам клиентов необходимо иметь возможность доступа к списку устройств клиента.

Если вы планируете использовать имя изготовителя оборудования, серийный номер и модель, учитывайте следующие ограничения.

- Этот кортеж работает только для новых устройств (например, 4 КБ) и не поддерживается для хэшей 128B (RS2 и предыдущих устройств).

- При регистрации кортежа учитывается регистр, поэтому данные в файле должны совпадать с именами модели и производителя ***точно*** так же, как это обеспечивает поставщик OEM (поставщик оборудования).

Следуйте приведенным ниже инструкциям, чтобы добавить устройства в учетную запись клиента в центре партнеров.

1. В меню Центр партнеров выберите **Клиенты** , а затем выберите клиента, устройства которого требуется управлять.

2. На странице сведения о клиенте выберите **устройства**.

3. В разделе **применить профили к устройствам** выберите **Добавить устройства**.

4. Введите имя для списка устройств и нажмите кнопку **Обзор** , чтобы отправить список клиента (в формате CSV-файла) в центр партнеров.

    >[!NOTE]
    >Вы должны получить этот CSV-файл с покупкой устройства. Если вы не получили CSV-файл, вы можете создать его самостоятельно, выполнив действия, описанные в разделе [Добавление устройств в Windows автопилот](https://docs.microsoft.com/windows/deployment/windows-autopilot/add-devices#collecting-the-hardware-id-from-existing-devices-using-powershell).  

5. Отправьте CSV-файл и нажмите кнопку **сохранить**.

Если при попытке отправить CSV-файл появляется сообщение об ошибке, проверьте формат файла. Можно использовать только хэш оборудования или имя изготовителя оборудования, серийный номер и модель (в этом порядке столбцов) или идентификатор продукта Windows. Вы также можете использовать файл Sample. csv, приведенный по ссылке далее, чтобы **Добавить устройства** для создания списка устройств.

CSV-файл должен выглядеть примерно так:

> **Серийный номер устройства, идентификатор продукта Windows, хэш оборудования, имя производителя, модель устройства**

> **{serialNumber},,, корпорация Майкрософт, портативный компьютер**

Обратите внимание, что "имя производителя" и "модель устройства" учитывают регистр.

Если вы не узнаете, какое значение следует указать для имени производителя и модели устройства, можно запустить его на устройстве, чтобы собрать правильные значения:

<pre><code>md c:\\HWID

Set-Location c:\\HWID

Set-ExecutionPolicy -Scope Process -ExecutionPolicy Unrestricted

Install-Script -Name Get-WindowsAutoPilotInfo

Get-WindowsAutoPilotInfo.ps1 -OutputFile AutoPilotHWID.csv -Partner -Force
</code></pre>

## <a name="windows-autopilot-eula-dismissal"></a>Закрытие лицензионного соглашения Windows для автопилота

### <a name="important-information"></a>ВАЖНАЯ ИНФОРМАЦИЯ

Windows автопилот позволяет настраивать настроенные установки Windows на устройствах, которыми вы управляете для клиентов. Если это разрешено клиентом, вы можете подавлять или скрывать определенные экраны настройки, которые обычно предоставляются пользователям при настройке Windows, включая экран подтверждения лицензионного соглашения (лицензионное соглашение).

С помощью этой функции вы соглашаетесь, что подавление или скрытие экранов, предназначенных для предоставления пользователям уведомления или принятия условий, означает, что вы получили достаточное согласие и авторизацию от клиента, чтобы скрыть термины, а также от имени клиента (будь то Организация или отдельный пользователь), согласие на любые уведомления и принимать любые условия, применимые к клиенту. Это включает согласие с условиями лицензии или уведомления, которое отображалось бы для пользователя, если не бы вы не отключили или не скрыли его с помощью этого средства. Ваш клиент не может использовать программное обеспечение Windows на этих устройствах, если ваш клиент не приобрел законным способом лицензию на программное обеспечение у Майкрософт или лицензированных дистрибьюторов.
