# <a name="explore-the-azure-and-cloud-products-benefits-area-in-partner-center"></a>Изучите область преимуществ Azure и облачных продуктов в центре партнеров

**Соответствующие роли**

- Глобальный администратор
- Администратор партнера MPN

В этой статье описываются конкретные преимущества Azure и облака, доступные партнерам в центре партнеров. Вы также узнаете, как активировать определенные типы Azure и облачные подписки.
 
## <a name="overview"></a>Обзор

В качестве партнера в Microsoft Partner Network вы можете получить доступ ко многим техническим и выходным преимуществам для различных областей. Если вы назначили одну из указанных выше ролей в центре партнеров, вы можете изучить многие из этих преимуществ непосредственно с панели мониторинга центра партнеров.

> [!NOTE]
> Чтобы получить доступ к области преимуществ центра партнеров, просто войдите на [панель мониторинга](https://partner.microsoft.com/dashboard) центра партнеров и выберите **MPN** в меню навигации слева. Затем выберите в меню пункт " **преимущества** ", а затем укажите конкретные преимущества или область, которую вы хотите изучить.

Одним из вариантов преимуществ, которые могут быть доступны вам, является **Azure & облачные продукты**. При выборе **azure & Cloud** из меню **преимуществ** центра партнеров отображается страница **Azure & Cloud Products** .

На этой странице представлены следующие сведения.

- Доступны различные преимущества Azure & облачных подписок на основе лицензий, подписки или подписки на основе использования.
- Количество лицензий, назначенных для каждого продукта
- Тип предложения, который был подготовлен для вас конкретным продуктом льготы
- Тип преимущества (от ядра или добавочного)
- Дата истечения срока действия конкретного преимущества

:::image type="content" source="images/benefits/mpn-benefits-azure-cloud-1.png" alt-text="На странице Azure and Cloud Products перечислены преимущества подписки, доступные вам." border="false":::

## <a name="usage-based-vs-license-based-subscriptions"></a>Сравнение использования и подписок на основе лицензий

На странице **Azure & Cloud Products** перечислены два типа преимуществ подписки, которые подготавливаются вам:
- Подписки на основе использования (Microsoft Azure кредита) 
- Подписки на основе лицензий (например, Office 365)

### <a name="usage-based-subscriptions-with-two-types-of-microsoft-azure-credit"></a>Подписки на основе использования с двумя типами Microsoft Azure кредита

Существует два типа подписок на основе использования, которые можно получить в зависимости от действующих предложений, доступных для вашего Microsoft Partner Network членства. К таким активным предложениям могут относиться Microsoft Action Pack, серебряные компетенции или компетенции категории Gold. 

Два типа подписок на основе использования, которые могут быть подготовлены для этих предложений, включают *ежемесячный кредит Azure* и *(или) Azure массовый (ежегодный) кредит*:

- **Подготовка для ежемесячного кредита Azure:** Это может быть нечто вроде $100/month (или эквивалент в местной валюте).<br/>
*и (или)*
- **Подготовка для Azure (ежегодно) кредита:** Это может выглядеть примерно так: $6000/year, $12000/year и/или $13000/year (или каждый из них эквивалентен в местной валюте).

Чтобы активировать Microsoft Azureную подписку в центре партнеров, необходимо назначить доступ пользователю в пределах компании (или гостевого пользователя). У пользователя уже должна быть запись пользователя, созданная в клиенте Azure Active Directory. Затем выполните следующие действия:

1. Если вы уверены, что у пользователя есть запись пользователя в клиенте Azure AD, войдите на [панель мониторинга](https://partner.microsoft.com/dashboard)центра партнеров.

1. Выберите **MPN** в меню навигации слева, а затем воспользуйтесь **преимуществами**, а затем — **Azure и Cloud**.

1. Щелкните стрелку вниз справа, **Просмотрите столбец сведений** для подписки на основе использования, которую необходимо активировать. 

1. Откройте область **назначения пользователей** на странице центра партнеров **Azure и облачные продукты** (см. ниже). Затем выберите параметр **назначить пользователя** , чтобы назначить этому пользователю нужное преимущество подписки на основе использования. 

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-2.png" alt-text="Страница &quot;Azure и облачные продукты&quot;, в которой отображается область назначения пользователей." border="false":::

- При назначении ежемесячного кредита Azure примерно $100 в месяц (или эквивалента в местной валюте) Обратите внимание, что в любое время вы можете добавлять или удалять пользователей из этой подписки.

- Обратите внимание, что при назначении Azure "групповое (ежегодное)", параметр " **Удалить** " недоступен в центре партнеров. Вместо этого необходимо переместить неоптовый кредит на кого-то другой в течение года регистрации. Кредитную подписку Azure (ежегодно) также нельзя сочетать с другими предложениями лицензирования, предоплатой или кредитным предложением (например, Microsoft Соглашение Enterprise, поставщиком облачных решений, Azure в открытых Azure Pass, Microsoft Action Pack и т. д.).

### <a name="license-based-subscriptions"></a>Подписки на основе лицензии

В разделе подписки на основе лицензий на странице Azure и облачные продукты вы найдете все облачные продукты, для которых требуется ключ продукта для активации, например Microsoft Office 365, Microsoft Dynamics, Microsoft Intune, Azure AD Basic, Power BI и т. д. Эти продукты предлагаются в зависимости от типа активного предложения, доступного вам как к партнеру в Microsoft Partner Network. К таким предложениям могут относиться Microsoft Action Pack, серебристое членство в компетенции или членство категории Gold.

Чтобы активировать подписку на основе лицензий, выполните следующие действия. 

1. На странице Azure и облачные продукты вы найдете конкретную подписку. 

1. Щелкните стрелку раскрывающегося списка справа, чтобы **Просмотреть столбец сведений** для этой подписки. 

1. Ознакомьтесь с инструкциями на следующей странице. Это позволит узнать больше о том, как активировать ключи. 

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-3.png" alt-text="Страница Azure и облачные продукты с подробными сведениями о просмотре." border="false":::


1. Когда вы будете готовы активировать ключ, выберите параметр **Пошаговое руководство по активации** , который находится в нижней части этой страницы.
   - Если вы хотите активировать новую подписку или ваша существующая подписка отключена или просрочена, откройте **частный или режиме инкогнито браузер**, скопируйте и вставьте следующую ссылку и следуйте инструкциям:

      [https://signup.microsoft.com/productkeystart](https://signup.microsoft.com/productkeystart)
      
   - Если вы хотите продлить дату окончания или добавить дополнительные рабочие места или лицензии в существующую подписку, скопируйте и вставьте приведенную ниже ссылку в **закрытом или режиме инкогнито окне браузера** и следуйте инструкциям:
 
      [https://admin.microsoft.com/AdminPortal/Home#/subscriptions](https://admin.microsoft.com/AdminPortal/Home#/subscriptions)

> [!Note]
> После активации ключа нельзя активировать один и тот же ключ еще раз. Также обратите внимание на следующие важные моменты.
>
> - Убедитесь, что вам нужно добавить дополнительные лицензии (рабочие места), сколько и не следует расширять конечную дату. Если вы решили добавить дополнительные лицензии (рабочие места) и **не** продлите дату окончания, система предоставит вам менее 12 месяцев активной подписки. 
>
> - Убедитесь, что вы вошли в систему с правильными учетными данными и активируете ключи в нужном клиенте Azure Active Directory (AAD). Всегда выполните вход с помощью закрытого или режиме инкогнито окна браузера, чтобы активировать ключи в правильном клиенте.
>
> - Дата окончания срока действия льготы, показанная в центре партнеров Майкрософт, указывает последнюю дату для использования прав на использование этого преимущества. Эта дата может отличаться от даты окончания подписки, указанной на портале центра администрирования M365. Это может быть вызвано несколькими факторами, например датой активации преимущества, числом лицензий, связанных с ключом, а также лицензиями, добавленными или расширенными на портале администрирования.

## <a name="extend-the-end-date-or-add-more-licenses-to-an-existing-subscription"></a>Расширение даты окончания или Добавление лицензий в существующую подписку

Выполните следующие действия, чтобы добавить дополнительные лицензии или продлить дату окончания для существующей подписки.

1. Скопируйте приведенную ниже ссылку и вставьте ее в **частное или режиме инкогнито окно браузера**.

   [https://admin.microsoft.com/AdminPortal/Home#/subscriptions](https://admin.microsoft.com/AdminPortal/Home#/subscriptions)

1. При входе на **портал центра администрирования M365** убедитесь, что вы входите в систему с помощью правильного клиента Azure Active Directory. Затем выберите в списке конкретную облачную подписку и выполните одно из следующих действий (чтобы продлить подписку или добавить дополнительные лицензии):

   - Чтобы продлить существующую подписку, выберите параметр **расширить конечную дату** в области лицензии. Увеличение даты окончания действия подписки увеличит время ее существования, сохраняя количество лицензий или рабочих мест. 

   - Чтобы добавить дополнительные лицензии (рабочие места) в существующую подписку, выберите параметр **Добавить лицензии** в области лицензия. Убедитесь, что не выбран параметр **Добавить лицензии** , если также требуется продлить существующую подписку. Выбрав **Добавить лицензии** , вы уменьшите длительность подписки и увеличите число лицензий в подписке.

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-4.png" alt-text="Страница Azure и облачные продукты, в которой показано, как добавить лицензии." border="false":::

1. На боковой панели выберите **активировать ключ продукта**. 

1. Введите ключ продукта, а затем нажмите кнопку **проверить** , чтобы проверить основные сведения. 

1. Нажмите кнопку **активировать** , чтобы завершить процесс.

   > [!NOTE]
   > Каждый ключ продукта предоставляет пользователям 12 месяцев обслуживания с даты активации ключа продукта. Тем не менее, если у существующих пользователей есть дата раннего продления, то для упрощения управления подписками на основе одной конечной даты для всех пользователей будут отображаться средние значения для двух дат. Во время вычисления время не добавляется или не теряется. Вычисление среднего значения из двух дат:
   >
   > **Вычисление:** \[ A (начальные пользователи) x B (время, оставшееся на подписку) \+ C (новые пользователи) x (12 месяцев) \] Общее число пользователей = новое оставшееся время.

## <a name="activate-a-subscription-for-the-first-time"></a>Активация подписки в первый раз

Выполните следующие действия, если вы активируете подписку в первый раз. Вы также можете использовать эти действия, если подписка находится в отключенном состоянии или с истекшим сроком действия.

1. Скопируйте приведенную ниже ссылку и вставьте ее в **частное или режиме инкогнито окно браузера**.
   
   [https://signup.microsoft.com/productkeystart](https://signup.microsoft.com/productkeystart)
1. Убедитесь, что вы вошли в правильную учетную запись клиента Azure Active Directory. После входа проверьте сведения о клиенте.

   :::image type="content" source="images/benefits/mpn-benefits-azure-cloud-5.png" alt-text="Страница Azure и облачные продукты, на которой показано, куда добавить ключ." border="false":::

1. Введите 25-значный ключ, полученный в центре партнеров, а затем нажмите кнопку **Далее**.

2. Проверьте отображаемые сведения о продукте, число лицензий и длительность подписки. Когда будете готовы, нажмите кнопку **активировать** , чтобы активировать подписку.

   Недавно активированная подписка будет доступна на вкладке **продукты** центра администрирования M365.

## <a name="licensing-scenarios"></a>Сценарии лицензирования

В этом разделе описаны несколько примеров сценариев лицензирования для подписок, связанных с Azure и облачными продуктами. В этих сценариях вы узнаете, как рассчитываются даты окончания подписки.

### <a name="scenario-1"></a>Сценарий 1

Если в подписке активна лицензия 100 и вы пытаетесь продлить дату окончания с помощью ключа с 25 лицензиями, Дата окончания будет продлена на три (3) месяца для подписки с 100.

### <a name="scenario-2"></a>Сценарий 2

Если в подписке активна лицензия 100 и вы пытаетесь продлить дату окончания с помощью ключа с лицензией 100, Дата окончания будет продлена на 12 месяцев.

### <a name="scenario-3"></a>Сценарий 3

Если у вас есть пять (5) лицензий, активных в подписке, и вы пытаетесь продлить дату окончания с помощью ключа с 10 лицензиями, Дата окончания будет продлена на 24 месяца для подписки с пятью (5) лицензиями (или местами).

### <a name="scenario-4"></a>Сценарий 4

Если вы пытаетесь продлить дату окончания с помощью ключа с более чем 3 раза активными лицензиями или местами, появится следующее сообщение об ошибке. Это сообщение обработано, так как Дата окончания подписки может быть продлена не более чем на 3 года.
