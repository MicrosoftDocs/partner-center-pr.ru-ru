# <a name="new-commerce-experience-in-csp---azure-billing"></a>Новый интерфейс для коммерческих приложений в CSP. Выставление счетов в Azure 

**Соответствующие роли:**

- Администратор выставления счетов
- Агент администрирования
- Глобальный администратор


Выставление счетов в рамках плана Azure — это простой инструмент выставления счетов за счет использования согласованной даты выставления счетов и расчетного периода на основе календарного месяца. Дополнительные сведения о платформе выставления счетов см. в статье [Общие сведения о выставлении счетов](billing-basics.md).

## <a name="summary-of-billing-essentials"></a>Основные сведения о выставлении счетов

- **Дата выставления счета-фактуры**. Счет-фактура и файл выверки будут доступны с помощью API или Панели мониторинга Центра партнеров в полночь по UTC.

- **Период выставления счетов**. Период выставления счетов определяется как за календарный месяц, например 1.10–31.10 или 1.11–30.11.

- **Периоды начисления оплаты**: Плата будет начисляться за календарный месяц. Например, если партнер, выставляющий счета, добавляет службы Azure через план Azure 15.10, а клиент начинает использование служб Azure с 15.10, то партнер получит счет-фактуру или файл выверки 8.11 по использованию клиентом служб в течение периода обслуживания 15.10–31.10. Счет за следующий месяц, который будет создан на 8.12, содержит все расходы за период обслуживания 1.11–31.11.

- **Срок оплаты счета-фактуры**. В течение 60 дней.

- **Валюта счета**. Партнеры будут по-прежнему получать счета-фактуры в назначенной валюте страны клиента. Например, если партнер в Ирландии выставляет счета клиентам в Великобритании, Норвегии и Германии, то партнер будет получать счет-фактуру или файл выверки в фунтах стерлингов (GBP), норвежских кронах (NOK) и евро (EUR).

- **Поощрения для партнеров**. Оплачено 45 дней с конца месяца счета-фактуры.

##  <a name="access-your-invoices-and-reconciliation-files"></a>Доступ к своим счетам и файлам выверки

Глобальный администратор или администратор выставления счетов для вашей компании получат электронное сообщение, когда счет-фактура будет готов к просмотру. 

**Доступ к счету и файлу выверки**

1. Выполните вход в [Панель мониторинга](https://partner.microsoft.com/en-us/dashboard/) Центра партнеров.

2. В меню Центра партнеров выберите **Billing** (Выставление счетов).

3. Выберите вкладки **Повторяемый** и **Единоразовый**, затем выберите интересующую вас валюту.

![выставление счетов](images/azure/billing1.png)

4. Выберите **счет** или **файл выверки**.  

Чтобы просмотреть предыдущие счета-фактуры и файлы выверки, разверните строку "История счетов" ниже.


## <a name="understanding-usage-data"></a>Основные сведения о данных об использовании 

1. План Azure — это корень или контейнер верхнего уровня для данных об использовании. Все данные об использовании привязываются к одному плану Azure. 

2. В плане может быть одна или несколько подписок Azure. Это контейнеры, используемые для управления ресурсами и развертывания. 

3. В подписку добавляются группы ресурсов, чтобы группировать ресурсы. Каждый ресурс развертывается в одной группе ресурсов. 

4. К примерам ресурсов относятся виртуальные машины и учетные записи хранения. 

5. Ресурсы выдают единицы измерения. Единицы измерения — это измерения потребления ресурса, и один ресурс может выдавать данные об использовании нескольких единиц измерения. Единицы измерения определяются идентификаторами ProductId, SKUId и AvailabilityId. 

### <a name="heirarchy-of-subscription-resource-groups-and-metering"></a>Иерархия групп ресурсов и контроля потребления ресурсов в подписке

**Учетная запись Azure (арендатор)**

- Подписка А
    - ГруппаРесурсов 1
        - Виртуальная машина (ресурс)
            - Единица измерения вычислений
        - Виртуальная сеть (ресурс)
            - Без оплачиваемых единиц измерения

    - ГруппаРесурсов 2
        - Виртуальная машина (ресурс)
            - Единица измерения компьютеров
        - Управляемый диск SSD (цен. категория "Премиум") (ресурс)
            - Единица измерения емкости хранилища
            - Единица измерения операций хранилища

- Подписка Б — ГруппаРесурсов1 — SQL Azure (ресурс) — Единица измерения DTU — VPN-шлюз (ресурс) — Единица измерения VPN-шлюзов

    - ГруппаРесурсов 2
        - Виртуальный сетевой интерфейс (ресурс)
            - Без оплачиваемых единиц измерения

## <a name="read-the-invoice"></a>Чтение счета-фактуры

1. Счет будет доступен не позднее 8 числа каждого месяца.

2. У партнеров есть 60 дней для предъявления платежа.

3. Расчетный период охватывает заданный календарный месяц, например 1.10–31.10.

4. Плата взимается в процентах от корректировок (сумма — это доля от "Партнерского кредита на управляемые службы").

5. Чтобы получить дополнительные сведения о выставлении счетов, см. файл выверки счета-фактуры и файл ежедневного использования.

![счет-фактура](images/azure/invoice1.png)

## <a name="read-the-invoice-reconciliation-file"></a>Чтение файла выверки для счета

1. Для каждого плана Azure и единицы измерения файл выверки может содержать до двух строк выставления счетов.

2. Если для единицы измерения действует какая-либо скидка или кредит (например, скидки на основе категории или партнерский кредит за управляемые службы) на протяжении всего календарного месяца, то файл выверки будет содержать только одну строку выставления счетов. Столбец **PriceAdjusmentDescription** будет учитывать скидку или полученный кредит.

3. Если для определенной единицы измерения нет ресурсов, для которых действует скидка или партнерский кредит, то файл выверки будет содержать только одну строку выставления счетов, а действительной ценой за единицу будет розничная цена (которая будет ценой за единицу).

4. Если для единицы измерения или выдающих ее ресурсов предоставлен **партнерский кредит за управляемые службы** за часть месяца, то файл выверки будет содержать две строки выставления счетов. Одна строка будет представлять дни, в которых счетчик действует, а вторая — дни, в которые счетчик не применяется. 

## <a name="read-the-daily-usage-file"></a>Чтение файла ежедневного использования

- Счетчики подписки в плане Azure оцениваются и распределяются ежедневно. 

- **Партнерский кредит для управляемых служб** определяется и применяется ежедневно.

- Каждый счетчик подписки будет иметь строку для каждого дня месяца, когда служба использовалась.

- См. пример ниже.

  - Счетчик действует для **партнерского кредита за управляемые службы** за период с 01.07 по 03.07 (обратите внимание, что цена за единицу является розничной ценой за вычетом партнерского кредита).

   - Счетчик не действует для **партнерского кредита за управляемые службы** за период с 04.07 по 07.07 (обратите внимание, что цена за единицу является розничной ценой).

    - Счетчик действует для **партнерского кредита за управляемые службы** за период с 08.07 по 31.07 (обратите внимание, что цена за единицу является розничной ценой за вычетом партнерского кредита).

## <a name="invoice-in-customer-currency"></a>Счет-фактура в валюте клиента 

Плата за службы Azure в плане Azure будет взиматься в долларах США (USD) и оплачиваться в валюте назначенной страны клиента. Если счет выставляется не в долларах США (USD), то на последней странице счета будет указан используемый курс валют. Курс валют определяется за месяц и применяется к следующему счету. Полный список валют см. в [New commerce offers country availability and customer currency matrix](https://query.prod.cms.rt.microsoft.com/cms/api/am/binary/RE3Qn1V) (Таблица валют клиентов и доступности новых коммерческих предложений в разных странах). 

Корпорация Майкрософт будет использовать данные [Thomson Reuters](https://developers.thomsonreuters.com/content/wm-company) при определении курса валют, используемого для определения и конвертации валюты выставления счетов. Курсы валют будут обновляться и доступны за день до первого дня месяца, для которого они применяются.

**Пример**:  Плата за использование в период обслуживания с 1 по 31 августа будет взиматься по курсу валют, опубликованному 31 июля. Эта плата отобразится в сентябрьском счете-фактуре, и курс валют будет указан на последней странице счета-фактуры. 

 
## <a name="azure-reservations"></a>Резервирования Azure 

В случае приобретения [резервирования Azure](https://docs.microsoft.com/partner-center/azure-reservations) с помощью плана Azure, сначала можно будет выбрать разовое выставление счета в Центре партнеров. Ежемесячное выставление счетов доступно на портале Azure. Ежемесячное выставление счетов будет доступно в Центре партнеров позже. 

## <a name="azure-spending"></a>Расходы на Azure 

Существующие возможности Azure по контролю расходов обновлены для поддержки выставления счетов по новым планам Azure в Центре партнеров. Это дает партнерам следующие возможности:

- просмотр, получение оповещений об установленном бюджете на уровне клиента и управление этими оповещениями; 

- просмотр общих прогнозируемых расходов на план Azure (с разбивкой на уровне ресурса и единицы измерения).

Так как модель выставления счетов для служб Azure в плане Azure — это использование служб после оплаты, чтобы избежать выставления счета, размер которого превышает ожидаемый, партнеры могут формировать месячный бюджет и следить за ростом использования служб. Бюджет может быть применен к одному клиенту или нескольким клиентам одновременно.
