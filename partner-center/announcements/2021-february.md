## <a name="put-qualification-apis-are-retired-get-to-retire-in-may"></a><a name="14"></a> Поддержка API-интерфейсов PUT для проверки соответствия требованиям прекращена, поддержка API-интерфейсов GET будет прекращена в мае

### <a name="categories"></a>Категории

- Дата: 25 февраля 2021 г.
- Характеристики

### <a name="impacted-audience"></a>Затронутая аудитория

Партнеры, продающие предложения для образовательных учреждений, некоммерческих организаций и предложения GCC в рамках программы "Поставщик облачных решений" и использующие API Центра партнеров.

### <a name="details"></a>Сведения

Это объявление дополняет сведения об [усовершенствованиях в Центре партнеров, выпущенных в декабре](2020-december.md#1). В рамках этого выпуска были развернуты новые API-интерфейсы GET и POST для проверки соответствия требованиям, в результате чего **поддержка существующих API-интерфейсов PUT для проверки соответствия требованиям была прекращена, а поддержка аналогичных API-интерфейсов GET будет прекращена 4 мая 2021 г.** Как только это произойдет, вам нужно будет перейти на новые API-интерфейсы POST Центра партнеров, чтобы приобретать предложения для образовательных учреждений. Аналогичным образом вам нужно будет использовать новый API-интерфейс GET для проверки соответствия требованиям, чтобы приобретать предложения для некоммерческих организаций и GCC, для которых выполнены предварительные требования.

### <a name="next-steps"></a>Дальнейшие действия

- **Выполните обновление до новых API** для успешного и своевременного перехода.
- **Ознакомьтесь со сведениями об изменениях API-интерфейсов в Центре партнеров и руководстве** в ресурсах по подготовке к работе: [Улучшения процесса проверки клиентов в Центре партнеров.](https://partner.microsoft.com/resources/collection/partner-center-edu-validation-enhancements#/)
- **Передайте эти сведения соответствующим командам** в своей организации и торговым посредникам, чтобы помочь им подготовиться к изменениям.

- Дата: 25.02.2021
- Характеристики

Партнеры с решениями, готовыми к совместной продаже.

### <a name="summary"></a>Итоги

Теперь партнеры могут приглашать продавцов Майкрософт после заключения партнерской сделки совместной продажи, чтобы совместно вести работу в рамках этой сделки.

### <a name="details"></a>Сведения

Теперь у партнеров есть возможность приглашать специалистов Майкрософт по продажам после создания сделки совместной продажи под руководством партнера, чтобы совместно вести работу в рамках этой сделки. Сведения о сделках не предоставляются отделу продаж Майкрософт, пока партнер явно не обратится за помощью к специалисту Майкрософт по продажам. Этот сценарий также подходит для поощрения специалистов Майкрософт по продажам.

Обратите внимание, что сделки совместной продажи под руководством партнера можно преобразовать в активную совместную продажу в любой момент до закрытия сделки.

## <a name="changes-to-the-cloud-solution-provider-csp-customer-onboarding-requirements"></a><a name="10"></a>Изменения в требованиях к подключению клиентов для участников программы "Поставщик облачных решений" (CSP)

### <a name="categories"></a>Категории

- Дата: 02.04.2021
- Расширенное обеспечение безопасности и соответствия требованиям

### <a name="summary"></a>Итоги

Так как мы стремимся поддерживать доверительные отношения с партнерами и клиентами при оказании им помощи в ведении бизнеса, мы будем запрашивать дополнительные сведения о клиентах с 25 марта 2021 г.

### <a name="impacted-audience"></a>Затронутая аудитория

Партнеры с прямым выставлением счетов по программе CSP и косвенные поставщики с новыми или существующими клиентами в странах, приведенных в следующем разделе.

### <a name="details"></a>Сведения

Корпорация Майкрософт работает на основе принципов доверия, Мы стремимся обеспечить соответствующий требованиям, безопасный и защищенный метод проверки клиентов для работы с клиентскими подписками по программе "Поставщик облачных решений" (CSP). 25 марта 2021 г. мы представим улучшения API и пользовательского интерфейса Центра партнеров, которые повлияют на работу партнеров, соответствующих обоим следующим критериям:

1. партнер, напрямую получающий счета от Майкрософт (то есть партнер с прямым выставлением счетов или косвенный поставщик);

2. партнер, ведущий бизнес с новыми или существующими клиентами в таких странах:

    - Таиланд
    - Вьетнам
    - Турция
    - Польша
    - ЮАР
    - Индия
    - Бразилия
    - Ирак
    - Мьянма
    - Южный Судан
    - Саудовская Аравия
    - ОАЭ
    - Венесуэла

Партнеры, которые удовлетворяют критериям, должны будут сообщить нам идентификатор регистрации компании клиента (также называемый ИНН организации клиента) и его номер телефона при следующем обновлении или создании подписки для такого клиента. Такие партнеры теперь также могут при желании указать отчество клиента.

Обратите внимание, что при добавлении идентификатора регистрации компании следует использовать идентификационный номер налогоплательщика вашего предприятия, а не личный идентификатор клиента.

Партнеры, которые ведут бизнес с новыми или существующими клиентами в следующих странах, уже были подключены после предыдущего выпуска в ноябре 2020 г.

- Армения
- Азербайджан
- Беларусь
- Венгрия
- Казахстан
- Киргизстан
- Молдова
- Россия
- Таджикистан
- Украина
- Узбекистан

Партнеры с клиентами в других странах смогут при желании указывать идентификатор регистрации компании, номер телефона и отчество или второе имя клиента с конца марта 2021 г.

## <a name="now-live-docusign-migration-to-adobe-sign-for-partners-under-microsoft-partner-agreements-mpas"></a><a name="9"></a>Уже доступно. Переход с DocuSign на Adobe Sign для партнеров, подписавших Соглашение с партнером Майкрософт (MPA)

### <a name="categories"></a>Категории

- Дата: 16.02.2021 г.
- Характеристики

### <a name="summary"></a>Сводка

Корпорация Майкрософт переведет обработку всех электронных подписей с DocuSign на Adobe Sign.

### <a name="impacted-audience"></a>Затронутая аудитория

Существующие прямые и косвенные партнеры в рамках программы "Поставщик облачных решений" (CSP), заключившие MPA.

### <a name="details"></a>Сведения

В феврале 2021 г. корпорация Майкрософт начнет переводить обработку всех электронных подписей с DocuSign на Adobe Sign для партнеров в рамках CSP.

Этот процесс будет выполняться максимально удобно для вас. После миграции вы будете получать сообщение электронной почты от adobesign@adobesign.com (а не от DocuSign) каждый раз, когда требуется электронная подпись. В этом сообщении будет ссылка на веб-страницу Adobe Sign, где нужно будет поставить подпись. Партнерам корпорации Майкрософт не придется повторно подписывать существующие соглашения, а только будущие соглашения с партнерами по сбыту. Партнерам в Украине, России и Казахстане нужно будет ежегодно физически или в электронной форме повторно подписывать поправки.

Пример сообщения электронной почты для запрашивания подписи в Adobe Sign можно найти в [коллекции ресурсов](https://partner.microsoft.com/resources/detail/adobe-sign-signature-request-email-pdf).

Для оптимального удобства работы выполните следующие действия:

1. Добавьте adobesign@adobesign.com в список надежных отправителей, чтобы избежать отправки сообщений из этой учетной записи непосредственно в папку нежелательной почты.
2. Выполните следующие действия в сотрудничестве с ИТ-отделом:
    - Добавьте адрес электронной почты adobesign@adobesign.com в список надежных отправителей, чтобы он не блокировался предварительно установленным правилом предотвращения фишинга.
    - Обновите существующие политики безопасности, чтобы все необходимые получатели могли подписывать документацию в рамках корпоративной лицензии на Adobe Sign.

Ресурсы по вопросам подключения, функциональности и бизнес-процессов можно найти на [странице перехода на Adobe Sign для партнеров](https://aka.ms/eSignature/External).

Adobe Sign — это предпочитаемое корпорацией Майкрософт решение для обработки электронных подписей (ESS). Переход на Adobe Sign сделает работу с электронными подписями безопасной и эффективной, обеспечив множество преимуществ для наших клиентов и партнеров.

Дополнительные сведения об электронном подписывании документов и делегировании электронной подписи см. в следующих руководствах:

- [Электронное подписывание документа | Руководства по Adobe Sign](https://helpx.adobe.com/sign/how-to/adobe-for-signers.html?playlist=/ccx/v1/collection/product/sign/segment/designer/explevel/beginner/applaunch/continuinged/collection.ccx.js&ref=helpx.adobe.com)
- [Делегирование прав на подписывание документа другому лицу | Руководства по Adobe Sign](https://helpx.adobe.com/sign/how-to/use-the-delegator-role.html?playlist=/ccx/v1/collection/product/sign/segment/designer/explevel/beginner/applaunch/orientation/collection.ccx.js&ref=helpx.adobe.com)

## <a name="reminder-introducingapithrottlingtopartners-calling-partner-centerapis"></a><a name="8"></a> Напоминание. Представляем регулирование интерфейсов API для партнеров, вызывающих интерфейсы API Центра партнеров

### <a name="categories"></a>Категории

- Дата: 16.02.2021 г.
- Развитие бизнеса.

### <a name="summary"></a>Сводка

### <a name="details"></a>Сведения

С февраля 2021 г. корпорация Майкрософт реализует регулирование API-интерфейсов, чтобы обеспечить более стабильную производительность в течение определенного периода для партнеров, вызывающих интерфейсы API Центра партнеров. При регулировании количество запросов к службе в конкретный период ограничивается, чтобы предотвратить чрезмерное потребление ресурсов. Если порог регулирования превышен, в Центре партнеров ограничивается количество последующих запросов от этого клиента на определенный период.

**Преимущества для партнеров.**

Центр партнеров предназначен для обработки большого количества запросов, но когда несколько партнеров отправляют огромное количество запросов, регулирование поможет обеспечить оптимальную производительность и надежность для всех партнеров.  

- Регулирование гарантирует минимальное время простоя.
- Регулирование позволяет сократить количество запросов при их поступлении в большом объеме, помогая обеспечить стабильную производительность для всех партнеров.

Это объявление дополняет сведения об [усовершенствованиях в Центре партнеров, выпущенных в декабре](./2020-december.md#1). В рамках этого выпуска были развернуты новые API-интерфейсы квалификаций GET и POST. Как следствие, **поддержка существующих API-интерфейсов квалификаций PUT Центра партнеров будет прекращена до 25 февраля 2021, а GET — до 4 мая 2021 г.** К этому времени нужно будет перейти на новые API-интерфейсы POST Центра партнеров, чтобы приобрести предложения для образовательных учреждений, и новый API квалификаций GET для приобретения предложений для некоммерческих организаций и предложений GCC с предварительными квалификациями.

### <a name="next-steps"></a>Дальнейшие действия

- **Выполните обновление до новых API** для успешного и своевременного перехода.
- **Ознакомьтесь со сведениями об изменениях API-интерфейсов в Центре партнеров и руководстве** в ресурсах по подготовке к работе: [Улучшения процесса проверки клиентов в Центре партнеров.](https://partner.microsoft.com/resources/collection/partner-center-edu-validation-enhancements#/)
- **Передайте эти сведения соответствующим командам** в своей организации и торговым посредникам, чтобы помочь им подготовиться к изменениям.
