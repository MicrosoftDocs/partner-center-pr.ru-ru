## <a name="now-live-docusign-migration-to-adobe-sign-for-partners-under-microsoft-partner-agreements-mpas"></a><a name="9"></a>Уже доступно. Переход с DocuSign на Adobe Sign для партнеров, подписавших Соглашение с партнером Майкрософт (MPA)

### <a name="categories"></a>Категории

- Дата: 16.02.2021 г.
- Характеристики

### <a name="summary"></a>Сводка

Корпорация Майкрософт переведет обработку всех электронных подписей с DocuSign на Adobe Sign.

### <a name="impacted-audience"></a>Затронутая аудитория

Существующие прямые и косвенные партнеры в рамках программы "Поставщик облачных решений" (CSP), заключившие MPA.

### <a name="details"></a>Сведения

В феврале 2021 г. корпорация Майкрософт начнет переводить обработку всех электронных подписей с DocuSign на Adobe Sign для партнеров в рамках CSP.

Этот процесс будет выполняться максимально удобно для вас. После миграции вы будете получать сообщение электронной почты от adobesign@adobesign.com (а не от DocuSign) каждый раз, когда требуется электронная подпись. В этом сообщении будет ссылка на веб-страницу Adobe Sign, где нужно будет поставить подпись. Партнерам корпорации Майкрософт не придется повторно подписывать существующие соглашения, а только будущие соглашения с партнерами по сбыту. Партнерам в Украине, России и Казахстане нужно будет ежегодно физически или в электронной форме повторно подписывать поправки.

Пример сообщения электронной почты для запрашивания подписи в Adobe Sign можно найти в [коллекции ресурсов](https://partner.microsoft.com/resources/detail/adobe-sign-signature-request-email-pdf).

Для оптимального удобства работы выполните следующие действия:

1. Добавьте adobesign@adobesign.com в список надежных отправителей, чтобы избежать отправки сообщений из этой учетной записи непосредственно в папку нежелательной почты.
2. Выполните следующие действия в сотрудничестве с ИТ-отделом:
    - Добавьте адрес электронной почты adobesign@adobesign.com в список надежных отправителей, чтобы он не блокировался предварительно установленным правилом предотвращения фишинга.
    - Обновите существующие политики безопасности, чтобы все необходимые получатели могли подписывать документацию в рамках корпоративной лицензии на Adobe Sign.

Ресурсы по вопросам подключения, функциональности и бизнес-процессов можно найти на [странице перехода на Adobe Sign для партнеров](https://aka.ms/eSignature/External).

Adobe Sign — это предпочитаемое корпорацией Майкрософт решение для обработки электронных подписей (ESS). Переход на Adobe Sign сделает работу с электронными подписями безопасной и эффективной, обеспечив множество преимуществ для наших клиентов и партнеров.

Дополнительные сведения об электронном подписывании документов и делегировании электронной подписи см. в следующих руководствах:

- [Электронное подписывание документа | Руководства по Adobe Sign](https://helpx.adobe.com/sign/how-to/adobe-for-signers.html?playlist=/ccx/v1/collection/product/sign/segment/designer/explevel/beginner/applaunch/continuinged/collection.ccx.js&ref=helpx.adobe.com)
- [Делегирование прав на подписывание документа другому лицу | Руководства по Adobe Sign](https://helpx.adobe.com/sign/how-to/use-the-delegator-role.html?playlist=/ccx/v1/collection/product/sign/segment/designer/explevel/beginner/applaunch/orientation/collection.ccx.js&ref=helpx.adobe.com)

## <a name="reminder-introducingapithrottlingtopartners-calling-partner-centerapis"></a><a name="8"></a> Напоминание. Представляем регулирование интерфейсов API для партнеров, вызывающих интерфейсы API Центра партнеров

### <a name="categories"></a>Категории

- Дата: 16.02.2021 г.
- Развитие бизнеса.

### <a name="summary"></a>Сводка

Корпорация Майкрософт реализует регулирование API-интерфейсов, чтобы обеспечить более стабильную производительность в течение определенного периода для партнеров, вызывающих интерфейсы API Центра партнеров.

### <a name="impacted-audience"></a>Затронутая аудитория

Все партнеры, заключающие сделки в рамках программы "Поставщик облачных решений" (CSP).  

### <a name="details"></a>Сведения

С февраля 2021 г. корпорация Майкрософт реализует регулирование API-интерфейсов, чтобы обеспечить более стабильную производительность в течение определенного периода для партнеров, вызывающих интерфейсы API Центра партнеров. При регулировании количество запросов к службе в конкретный период ограничивается, чтобы предотвратить чрезмерное потребление ресурсов. Если порог регулирования превышен, в Центре партнеров ограничивается количество последующих запросов от этого клиента на определенный период.

**Преимущества для партнеров.**

Центр партнеров предназначен для обработки большого количества запросов, но когда несколько партнеров отправляют огромное количество запросов, регулирование поможет обеспечить оптимальную производительность и надежность для всех партнеров.  

- Регулирование гарантирует минимальное время простоя.
- Регулирование позволяет сократить количество запросов при их поступлении в большом объеме, помогая обеспечить стабильную производительность для всех партнеров.

## <a name="introducing-microsoft-viva-topics"></a><a name="7"></a>Представляем Microsoft Viva Topics

### <a name="categories"></a>Категории

- Дата: 12 февраля 2021 г.
- Характеристики

### <a name="affected-audience"></a>Затронутая аудитория

Все партнеры, заключающие сделки в рамках программы "Поставщик облачных решений" (CSP).

### <a name="details"></a>Сведения

Решение Microsoft Viva Topics с помощью средств искусственного интеллекта (ИИ) определяет базу знаний и экспертных навыков в вашей организации. На основе этих данных создаются курируемые общие темы, например относящиеся к продуктам, клиентам или проектам. Иммерсивные карточки и страницы тем обеспечивают доступ к базе знаний в контексте команд, проектов и документов. ИИ и сотрудники совместно совершенствуют базу знаний, чтобы вы могли ежедневно использовать ее в приложениях. 

Для работы с Viva Topics требуются лицензии на следующие продукты:   
- Microsoft 365 F1, F3, E3, A3, E5 или A5; 
- Office 365 F3, E1, A1, E3, A3, E5 или A5; 
- Microsoft 365 бизнес базовый, стандартный или премиум; 
- SharePoint K, план 1 или план 2. 

**Сведения о предложении**

В прейскурантах от 1 февраля в названия этих предложений будет добавлен текст "Topic Experiences".  При обновлении прейскурантов 1 марта в названиях предложений будет указано "Viva Topics".

### <a name="impacted-audience"></a>Затронутая аудитория

- Партнеры с предложениями, готовыми к совместной продаже.
- Партнеры с предложением, которое может быть предметом сделки, использующие решение Azure, которое соответствует требованиям программы поощрений, на коммерческой платформе.
- Все партнеры, использующие модуль клиентов по рекомендации в Центре партнеров.

### <a name="details"></a>Сведения

С началом нового года мы добавляем функции для улучшения и дополнения возможностей совместной продажи в Центре партнеров. Эти новые функции помогут вам ускорить ведение сделок, расширить возможности доступа продавцов и улучшить масштабирование бизнеса. Новые дополнения в модуле клиентов по рекомендации позволят привлекать больше клиентов благодаря сотрудничеству с другими партнерами или продавцами Майкрософт. При этом вы сможете управлять своим конвейером совместно с корпорацией Майкрософт в одном расположении.  

Новые возможности в модуле клиентов по рекомендации в Центре партнеров:

#### <a name="all-partners-using-the-referrals-module-in-partner-center"></a>Все партнеры, использующие модуль клиентов по рекомендации в Центре партнеров  

- [Создание новых подключений SMB.](https://docs.microsoft.com/partner-center/connect-with-your-customers) Совместная продажа и сотрудничество с продавцами Майкрософт для получения внутренних возможностей, связанных с клиентами SMB.

- [Расширение разрешений в модуле клиентов по рекомендации.](https://docs.microsoft.com/partner-center/permissions-overview#manage-referrals) Назначайте пользователям модуля клиентов по рекомендации разрешения администратора (доступ ко всем возможностям совместной продажи) или пользователя (доступ к конкретным возможностям совместной продажи для пользователей).

- [Перенос пользователей Partner Sales Connect.](https://docs.microsoft.com/partner-center/psc-to-pc#user-migration) Сопоставьте существующие разрешения пользователей Partner Sales Connect с соответствующими разрешениями администратора и пользователя в модуле клиентов по рекомендации в Центре партнеров, а затем массово примените новые разрешения к пользователям. Дополнительные сведения об управлении клиентами по рекомендации см. в [документации по разрешениям в модуле клиентов по рекомендации](https://docs.microsoft.com/partner-center/permissions-overview#manage-referrals).  

#### <a name="partners-with-co-sell-ready-offers"></a>Партнеры с предложениями, готовыми к совместной продаже.

- [Совместная продажа с другими партнерами.](https://docs.microsoft.com/partner-center/manage-co-sell-opportunities) Пригласите другого партнера из нашей высокоэффективной экосистемы партнеров для совместной продажи и расширьте клиентскую базу.  

#### <a name="partners-with-a-transactable-offer-using-an-azure-incentive-eligible-solution-in-the-commercial-marketplace"></a>Партнеры с предложением, которое может быть предметом сделки, использующие решение Azure, которое соответствует требованиям программы поощрений, на коммерческой платформе.

- [Использование автоматической регистрации сделки.](https://docs.microsoft.com/partner-center/register-deals) Сэкономьте время и обеспечьте точность, зарегистрировав предложения Azure, которые могут быть предметом сделки, размещенные на коммерческой платформе Майкрософт, с помощью флажка This deal/win has been or will be transacted through Azure Marketplace or AppSource (Эта потенциальная или заключенная сделка будет проведена посредством Azure Marketplace или AppSource). Если дата сделки верна, информация для регистрации сделки будет извлекаться непосредственно из коммерческой платформы Майкрософт.

### <a name="questions"></a>У вас появились вопросы?

Если вам нужна помощь с использованием каких-либо новых функций, ознакомьтесь с [разделом о клиентах по рекомендации](https://docs.microsoft.com/partner-center/referrals) в нашей базе знаний или в практических материалах по поддержке, доступных в [этой коллекции ресурсов по совместной продаже](https://aka.ms/CoSellExperience) (нужно выполнить вход).

Чтобы получить помощь, вы можете [отправить запрос в службу поддержки партнеров](https://partner.microsoft.com/support/?stage=1).

## <a name="deprecation-and-retirement-of-put-qualification-apis-for-the-education-customer-validation-process-by-february-25-2021-and-get-qualification-by-may-4-2021"></a><a name="1"></a> Объявление нерекомендуемыми и прекращение поддержки API-интерфейсов квалификации PUT для проверки соответствия требованиям клиентов из сферы образования до 25 февраля 2021 г. и квалификации GET до 4 мая 2021 г.

### <a name="categories"></a>Категории

- Дата: 4 февраля 2021 г.
- Характеристики

### <a name="impacted-audience"></a>Затронутая аудитория

Партнеры, продающие предложения для образовательных учреждений, некоммерческих организаций и предложения GCC в рамках программы "Поставщик облачных решений" и использующие API Центра партнеров.

### <a name="details"></a>Сведения

Это объявление дополняет сведения об [усовершенствованиях в Центре партнеров, выпущенных в декабре](https://docs.microsoft.com/partner-center/announcements/2020-december#1). В рамках этого выпуска были развернуты новые API-интерфейсы квалификаций GET и POST. Как следствие, **поддержка существующих API-интерфейсов квалификаций PUT Центра партнеров будет прекращена до 25 февраля 2021, а GET — до 4 мая 2021 г.** К этому времени нужно будет перейти на новые API-интерфейсы POST Центра партнеров, чтобы приобрести предложения для образовательных учреждений, и новый API квалификаций GET для приобретения предложений для некоммерческих организаций и предложений GCC с предварительными квалификациями.
