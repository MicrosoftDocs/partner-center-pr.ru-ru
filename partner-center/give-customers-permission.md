# <a name="give-customers-permission-in-partner-center-to-buy-their-own-products-or-services"></a>Предоставление клиентам доступа в центре партнеров для приобретения своих продуктов или услуг

**Относится к**

- Центр партнеров
- Партнеры в программе CSP

**Соответствующие роли**

- Агент по администрированию
- Агент по продажам

В этой статье показано, как партнер в программе поставщика облачных решений (CSP) может предоставить клиенту разрешение на приобретение некоторых собственных служб или ресурсов.

Партнеры в программе CSP часто используют центр партнеров и свой коммерческий магазин для приобретения решений и услуг для своих клиентов. Затем партнеры могут предоставить некоторым клиентам доступ к этим службам непосредственно из портал Azure.

Пример приведен ниже. Предположим, вы приобретаете подписку на план Azure для клиента в центре партнеров. Затем вы решите добавить другие ресурсы или службы в эту подписку от имени клиента. В этом случае вы можете добавить резервирование Azure в подписку клиента (например, добавить зарезервированные экземпляры виртуальных машин). Затем вы можете предоставить клиенту возможность дальнейшей самостоятельно подготавливать ресурсы резервирования Azure в портал Azure.

Теперь с помощью функции " **разрешения клиентов** " вы предоставляете пользователям дополнительные возможности самообслуживания с ресурсами Azure. Включив разрешения для клиента, вы разрешаете клиентам покупать собственные ресурсы (например, покупать собственные резервирования Azure).  

## <a name="overview-of-customer-permissions-in-partner-center"></a>Обзор разрешений клиента в центре партнеров

Используйте страницу **учетная запись** клиента, чтобы включить (или отключить) разрешения клиента. В настоящее время эта функция поддерживает:

- **Резервирование Azure:** Включение этого разрешения позволяет клиенту приобрести собственные резервирования Azure для определенной подписки Azure, которую вы приобрели для них.

Прежде чем включать разрешения клиента, обратите внимание на следующие важные моменты.

- По умолчанию разрешения клиента автоматически отключаются (отключены) в центре партнеров.

- Прежде чем включать (или отключать) разрешения для клиента, необходимо назначить роль агента администратора в центре партнеров.

  Партнеры, которым назначена роль агента продаж или агента службы поддержки, имеют доступ только для чтения и не могут включать или выключать разрешения клиента.

- Вы можете включить разрешения (включить) для любого выбранного клиента.

- Вы можете включить или отключить разрешения клиентов с помощью панели мониторинга центра партнеров или [API центра партнеров](https://docs.microsoft.com/partner-center/develop/manage-customers).

- После включения разрешений (включить) для конкретного клиента вы несете ответственность за оплату всех последующих покупок, сделанных этим клиентом. Если клиенты хотят обмениваться данными, отменять или возобновлять приобретенные заказы (или хотят изменить первоначальную область резервирования), они не смогут сделать это самостоятельно. Им необходимо попросить вас, как партнера, помочь им в обмене, отмене и продлении покупок или внести последующие изменения в область резервирования.  

- После включения разрешений для конкретного клиента вы **не** будете уведомлены о последующей покупке, сделанной клиентом.

- Последующие покупки, сделанные клиентом, появятся в центре партнеров вместе с любыми покупками, сделанными вами. Эти покупки можно найти на странице **истории заказов** клиента, на странице **резервирования** или в [**журнале действий**](activity-logs.md).

>[!NOTE]
> Сведения о ценах, которые будут оплачиваться клиентом, а также о том, как помочь клиентам в управлении покупками, см. в статье [Помогите клиентам управлять своими зарезервированными товарами](give-customers-permission.md#help-customers-manage-reservations-they-purchase).

## <a name="give-customers-permission-to-buy-their-own-azure-reservations"></a>Предоставление клиентам разрешения на приобретение собственных резервирований Azure

Резервирования Azure — это отличный способ приобрести службы Azure со скидкой. Дополнительные сведения о преимуществах резервирования Azure см. в статье [что такое резервирование Azure?](https://docs.microsoft.com/azure/cost-management-billing/reservations/save-compute-costs-reservations)

Теперь у вас есть возможность приобрести резервирование Azure от имени клиентов, как уже было сделано. Вы также можете предоставить клиентам разрешение на приобретение собственных резервирований Azure.

>[!NOTE]
> После предоставления клиентам разрешения на покупку собственных резервирований Azure Помогите им управлять любыми вашими резервированиями. Дополнительные сведения см. в статье [Помогите клиентам в управлении зарезервированными ими](give-customers-permission.md#help-customers-manage-reservations-they-purchase).

### <a name="to-enable-customers-to-buy-their-own-azure-reservations"></a>Предоставление клиентам возможности покупать собственные резервирования Azure

1. Убедитесь, что у клиента есть существующий план Azure или глобальная подписка Azure, которую вы приобрели от своего имени.

2. Убедитесь, что клиенту назначена роль **владельца** для этой подписки.

3. Включите разрешения клиента (включите **эту функцию)**, чтобы приобрести собственные резервирования Azure.

Каждый шаг отображается ниже.

### <a name="verify-the-customer-has-an-existing-azure-subscription"></a>Проверка наличия у клиента существующей подписки Azure

Прежде чем предоставить клиентам разрешение на приобретение собственных резервирований Azure, необходимо убедиться, что у клиента есть существующий план Azure или глобальная подписка Azure. Если клиент не отображает текущую подписку Azure в центре партнеров, необходимо приобрести подписку для них, прежде чем включать разрешения клиента.

- Чтобы узнать, есть ли у клиента подписка Azure, войдите на панель мониторинга центра партнеров, а затем выберите **CSP** , а затем — **Клиенты**. Выберите конкретного клиента из списка. Затем выберите **подписки** и найдите подписки на основе использования для плана Azure или глобального Azure.

- Если у клиента нет подписки Azure, вы можете купить ее. См. статью о [приобретении плана Azure](purchase-azure-plan.md).

### <a name="verify-the-customer-has-been-assigned-the-correct-role-in-azure"></a>Убедитесь, что клиенту была назначена правильная роль в Azure.

Убедившись, что у клиента есть подписка Azure, необходимо также убедиться в том, что ключевые пользователи, связанные с вашим клиентом, назначили вам правильную роль **владельца** для этой подписки Azure. Это доступ на основе ролей (RBAC). Клиент должен купить резервирование Azure для приобретенной подписки Azure.

Некоторые партнеры могли уже назначить роль **владельца** клиентам, желающим активно управлять и подготавливать свои ресурсы Azure. Если вы уже назначили клиенту состояние **владельца** для управления ранее приобретенными подписками, этот шаг можно пропустить.  

> [!IMPORTANT]
> Если клиенту не назначена роль **владельца** , он получит сообщение об ошибке в портал Azure не позволит им покупать резервирование Azure.

Чтобы убедиться, что клиенту назначена роль **владельца** для подписки Azure, выполните следующие действия.

1. Войдите на [панель мониторинга](https://partner.microsoft.com/dashboard) Центра партнеров.

2. Выберите **CSP**, затем **Клиенты** и выберите конкретного клиента.

3. Выберите **подписки** для этого клиента и укажите конкретную подписку Azure.

4. Нажмите кнопку **Управление** рядом с подпиской этого клиента. Это приведет к открытию [портал Azure](https://portal.azure.com/).

5. Чтобы назначить роль **владельца** конкретному пользователю, выполните следующие действия, [чтобы назначить пользователя администратором](https://docs.microsoft.com/azure/cost-management-billing/manage/add-change-subscription-administrator#to-assign-a-user-as-an-administrator).

### <a name="turn-on-or-turn-off-customer-permissions-to-purchase-their-own-azure-reservations"></a>Включение или отключение разрешений клиента для приобретения собственных резервирований Azure

Убедившись, что у клиента есть подписка Azure, а пользователям назначена роль **владельца** для этой подписки, вы можете включить разрешения для пользователей (включить). Эти действия также можно использовать для выключения (отключения) разрешений для клиентов. Вы можете включить или отключить разрешения клиента с помощью панели мониторинга центра партнеров или [API центра партнеров](https://docs.microsoft.com/partner-center/develop/manage-customers).

Чтобы включить (или отключить) разрешения клиента в центре партнеров, выполните следующие действия.

1. Войдите на [панель мониторинга](https://partner.microsoft.com/dashboard) Центра партнеров.

2. В меню навигации слева выберите **CSP**, а затем — **Клиенты**. Появится список клиентов.

3. Выберите конкретное имя клиента.

4. В меню клиент выберите пункт **учетная запись** . Откроется страница **учетная запись** клиента.

5. В нижней части страницы щелкните область **разрешения клиентов** .

   :::image type="content" source="images/give-customers-permission-reservations.png" alt-text="Разрешения клиента на странице учетной записи." border="true":::

6. В разделе **резервирования Azure**выберите параметр **Разрешить клиенту покупку** .

7. Чтобы включить разрешения для клиента, переместите переключатель рядом с этим параметром в положение **вкл** . Чтобы отключить разрешения клиента, передвиньте переключатель в положение **выкл** .

>[!NOTE]
> Сведения о том, что происходит при включении разрешений клиента для приобретения собственных резервирований Azure, см. [в разделе Обзор разрешений клиентов в центре партнеров](give-customers-permission.md#overview-of-customer-permissions-in-partner-center).
>
>При включении (или отключении) разрешений клиента журнал действий записывает каждое действие. (Этот журнал доступен при выборе значка шестеренки в верхней части панели мониторинга центра партнеров). При включении или отключении разрешений клиента действие будет отображаться как **Создание разрешений на приобретение клиента** или **Удаление разрешений на покупку клиента** в журнале действий.

## <a name="help-customers-manage-reservations-they-purchase"></a>Помогите клиентам управлять зарезервированными ими покупками

После предоставления клиентам разрешения на покупку собственных резервирований Azure вы можете помочь им лучше управлять любыми приобретенными ресурсами. Клиенты могут управлять многими аспектами резервирования Azure непосредственно из [портал Azure](https://portal.azure.com/). Им потребуется помощь в управлении многими другими аспектами резервирования Azure, которые они приобрели в вашей подписке CSP.  

Помогите клиентам узнать больше об управлении этими аспектами резервирования Azure:

- Цены, по которым клиенты будут платить за резервирование Azure
- Как клиенты могут оптимизировать использование резервирований Azure
- Что происходит, когда клиенты приобретают резервирование с общей областью?
- Что произойдет, если клиенты хотят изменить, отменить или продлить резервирование или изменить его область?

**Цены клиенты будут платить за резервирование.** Ваш клиент будет покупать резервирование Azure на основе подписки, которая ранее была приобретена для них в учетной записи выставления счетов партнеров CSP. Цена клиента для всех резервирований Azure, приобретенных на основе этой подписки, также задается вами. Эта цена может отличаться от цены в Интернете, которую клиент видит в портал Azure.

**Как клиенты могут оптимизировать использование резервирования.** Некоторые клиенты могут получить преимущества от изучения того, как оптимизировать использование резервирования или назначить начальную область резервирования во время их приобретения. Чтобы получить дополнительные сведения, попросите клиентов прочитать [Управление резервированиями для ресурсов Azure](https://docs.microsoft.com/azure/cost-management-billing/reservations/manage-reserved-vm-instance).

**Что происходит, когда клиент приобретает резервирование с общей областью?** Когда клиенты приобретают резервирование на основе предыдущей подписки CSP и намерены использовать общую область для этого резервирования, все скидки, предоставленные клиентом CSP, будут применяться для сопоставления всех подписок, приобретенных партнером CSP для этого клиента.

**Что делать, если клиенты хотят выполнить обмен, отменять или продлить приобретенную покупку или изменить начальную область резервирования?** Клиенты должны попросить партнера помочь им изменить начальную область резервирования. Им также требуется помощь партнера по обмену, отмене или продлению резервирования. Они не могут выполнять эти задачи с помощью резервирования на основе подписок, приобретенных для них партнером CSP.
